# Manage macOS with Ansible

Replace nix-darwin with an Ansible playbook for the MacBook Pro.
Follow the same flat-playbook pattern used for Arch Linux.
Keep the two playbooks independent — no shared package lists.

## Motivation

Nix-darwin manages the mac's development environment but adds complexity
without proportional benefit. Ansible already manages the Arch workstation
and handles the same categories of work: packages, dotfiles, directories,
and tool setup. One tool for both machines simplifies maintenance.

## File Structure

```
ops/mac/
  inventory              # localhost, ansible_connection=local
  playbook.yml           # Single flat playbook, no roles
  requirements.yml       # community.general collection
  README.md              # Bootstrap instructions
```

No tags. No roles. The playbook runs without `become: yes` at the
top level — Homebrew discourages root. Individual tasks elevate
privileges only when necessary.

## Package Split: Brew vs Mise

**Brew** installs system-level tools not tied to a language runtime:

| Category | Packages |
|----------|----------|
| Core CLI | `bat`, `curl`, `delta`, `fd`, `fzf`, `git`, `jq`, `less`, `ncdu`, `neovim`, `pigz`, `ripgrep`, `rsync`, `sqlite`, `stow`, `tmux`, `tree`, `unzip`, `vim`, `vifm`, `wget`, `zsh` |
| Cloud | `awscli`, `aws-vault` |
| Dev | `mise` |
| Casks | `1password`, `1password-cli`, `kitty` |

**Mise** installs runtimes, LSPs, formatters, and language-specific
tools. These are declared in `mise/.config/mise/config.toml` (stowed
into place) and hydrated by `mise install`:

| Category | Tools |
|----------|-------|
| Runtimes | `nodejs`, `python` |
| LSPs | `lua-language-server`, `typescript-language-server`, `htmx-lsp`, `ty` |
| Formatters | `prettier`, `stylua` |
| Other | `tree-sitter`, `tinty` (cargo backend) |

**Rule:** Brew never installs anything mise manages. Adding a new LSP
or formatter means editing `config.toml`, not the playbook.

## Stow Packages

The playbook stows 13 packages from `~/dev/dot_files/`:

```
bat, bin, claude, git, kitty, mise, npm, nvim, sql, tinty, tmux, vifm, zsh
```

Stow options: `--restow --no-folding --verbose=1`

Dropped from the nix-darwin list: `mpv` (macOS has QuickTime).

## Setup Tasks

After package installation, the playbook runs these tasks in order:

1. **Create directories** — `~/.ssh` (mode 700), `~/.tmux/plugins`,
   `~/dev`, `~/docs`, `~/pics`, `~/videos`
2. **Install oh-my-zsh** — Clone if `~/.oh-my-zsh` does not exist
3. **Stow dotfiles** — Restow 13 packages listed above
4. **Install tmux plugin manager** — Clone tpm to
   `~/.tmux/plugins/tpm` if not present
5. **Run mise install** — Hydrate runtimes, LSPs, formatters, and
   tinty from `config.toml`

## Theme Management: Tinty + fzf

Tinty works on macOS. The hooks for kitty, tmux, fzf, nvim, starship,
and vifm are cross-platform. Wayland-only hooks (waybar, fuzzel, mako)
fail silently when those binaries are absent.

The `set-color-scheme-preference.sh` hook needs a macOS adaptation.
Replace the `dconf` calls with `defaults write -g AppleInterfaceStyle`
to toggle macOS dark mode for apps that respect the system setting.

### Theme Picker

On Arch, the fuzzel-based `theme-menu.sh` provides a graphical theme
picker. On macOS, an fzf-based script replaces it.

Location: `bin/.bin/,mac-theme-menu.sh` (follows existing prefix
convention for platform-specific scripts).

Behavior: list tinty schemes through fzf with favorites shown first,
apply the selected scheme.

## Bootstrap Process

### Prerequisites (manual, one-time)

```bash
xcode-select --install
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
brew install git ansible
git clone <repo-url> ~/dev/dot_files
ansible-galaxy collection install -r ops/mac/requirements.yml
```

### Run

```bash
ansible-playbook -i ops/mac/inventory ops/mac/playbook.yml
```

### Post-playbook (manual, one-time)

- Open 1Password and sign in (needed for SSH agent)
- Run `tmux` then `prefix + I` to install tmux plugins
- Restart terminal to pick up zsh config

## What Gets Removed

After the Ansible playbook is validated:

- Delete `ops/nix/hosts/mac/` (mac-specific nix config)
- Delete `ops/nix/common/darwin.nix`
- Remove the mac output from `ops/nix/flake.nix`
- Remove nix-darwin and home-manager darwin references from flake inputs
  (if forge does not use them)
- Uninstall nix-darwin from the mac
- Uninstall nix from the mac

The NixOS configuration for forge remains untouched.

## Unchanged

- **Dotfile management** — Stow continues to manage all config files
- **1Password SSH integration** — Handled by stowed SSH/git config
- **Starship config** — Generated by tinty, not stowed
- **Arch playbook** — Independent; no changes needed
- **NixOS forge config** — Unaffected
